## 🏗️ Multi-Model Pipeline

**Swin-UNet** (context segmentation) → **YOLOv8** (small objects) → **GeoSAM** (refinement) → **Depth** → **Learned Fusion Orchestrator** → GeoPackage + COG export.

> **Key design**: The Orchestrator uses a **trained late-fusion model** (not hardcoded rules) to mediate between models — deciding acceptance, refinement, and class assignment.

## Quick Start

### 1. Setup Environment

```bash
# Create virtual environment
python3 -m venv venv
source venv/bin/activate

# Install dependencies
pip install -r requirements.txt
```

### 2. Data Layout

Place your files as follows:

```
data/
├── raw/                  # Government orthophotos: {village}_{tileid}.tif
├── pointcloud/           # LAS/LAZ files: {village}_{tileid}.las (optional)
├── dem/                  # DSM/DTM GeoTIFFs (optional)
├── meta/                 # Per-tile JSON metadata (optional)
├── annotations/          # Ground truth: {village}_groundtruth.gpkg
│   └── archive/          # Versioned backups
└── preprocessed/         # Generated by pipeline
```

**SVAMITVA Datasets** (download and place in `data/raw/`):

| Region | Link |
|--------|------|
| CG Training (Set 2) | https://svamitva.nic.in/DownloadPDF/TifFile/CG_Training_dataSet_2.zip |
| CG Training (Set 3) | https://svamitva.nic.in/DownloadPDF/TifFile/CG_Training_dataSet_3.zip |
| CG SHP Files | https://svamitva.nic.in/DownloadPDF/TifFile/CG_shp-file.zip |
| PB Training + SHP | https://svamitva.nic.in/DownloadPDF/TifFile/PB_training_dataSet_shp_file.zip |

### 3. SAM Checkpoint (for GeoSAM)

Download SAM ViT-H and place it at `models/sam_vit_h.pth`:

```bash
wget -P models/ https://dl.fbaipublicfiles.com/segment_anything/sam_vit_h_4b8939.pth
mv models/sam_vit_h_4b8939.pth models/sam_vit_h.pth
```

> If the checkpoint is missing, GeoSAM runs in **stub mode** returning deterministic masks.

### 4. Run the Pipeline

```bash
# Generate sample data for testing
bash scripts/create_sample_data.sh

# Step 1: Preprocess (variance filter + tiling)
bash scripts/run_preprocess.sh config/config.yaml

# Step 2: Train Swin-UNet (all folds)
for fold in 0 1 2 3 4; do
  bash scripts/run_train_swin.sh config/config.yaml $fold
done

# Step 3: Train Fusion Model
bash scripts/run_train_fusion.sh config/config.yaml

# Step 4: Run Inference
bash scripts/run_inference.sh config/config.yaml run_01

# Step 5: Export to GeoPackage + COG
bash scripts/export_to_ogc.sh run_01 config/config.yaml
```

### 5. CPU-Only / Debug Mode

```bash
# Single-tile debug on CPU
python src/inference/run_inference.py \
  --config config/config.yaml \
  --run-id debug_01 --debug --cpu-only

# Lite inference (ONNX, CPU only)
python src/inference/lite_infer.py \
  --config config/config.yaml \
  --tile data/preprocessed/tiles/512/some_tile.png
```

### 6. ONNX Export + Quantization

```bash
bash scripts/export_onnx.sh config/config.yaml
```

### 7. Smoke Test

```bash
python src/tests/smoke_test.py
```

## Architecture

```
Raw GeoTIFFs
    │
    ▼
┌─────────────────────┐
│  Variance Filter    │ → discard featureless tiles (JSON reasons)
│  Tiler (512/640px)  │ → overlapping tiles + metadata
└─────────────────────┘
    │
    ▼ (per tile, parallel)
┌──────────┐  ┌──────────┐
│ Swin-UNet│  │ YOLOv8n  │
│ (512px)  │  │ (640px)  │
└──────────┘  └──────────┘
    │              │
    ▼              ▼
┌──────────────────────────┐
│  LEARNED Fusion Model    │ ← trained on feature dumps + GT
│  (MLP / Transformer)    │
│  Outputs: accept/reject, │
│  class, refine?, conf    │
└──────────────────────────┘
    │
    ├─── Refine? ──→ GeoSAM (top-k boxes)
    ├─── Depth?  ──→ LAS/DSM-DTM
    │
    ▼
┌────────────────────┐
│ Topology Sanitizer │ → valid polygons
│ OGC Export         │ → .gpkg + COG .tif
└────────────────────┘
```

## CLI Reference

| Command | Description |
|---------|-------------|
| `python src/preprocess/variance_filter.py --config config.yaml` | Filter low-variance tiles |
| `python src/preprocess/tiler.py --config config.yaml` | Generate tiles |
| `python src/train/train_swin.py --config config.yaml --fold 0` | Train Swin-UNet |
| `python src/train/train_fusion.py --config config.yaml --generate-dataset` | Create fusion dataset + train |
| `python src/inference/run_inference.py --config config.yaml --run-id X` | Full inference |
| `python src/inference/lite_infer.py --config config.yaml --tile <path>` | Lite CPU inference |
| `python src/export/export_ogc.py --input output/X/tiles --out output/X/final.gpkg` | Export |

All commands support `--debug` and `--cpu-only` flags.

## Configuration

All parameters are in `config/config.yaml`. Key defaults:

| Parameter | Default | Description |
|-----------|---------|-------------|
| `V_GLOBAL_THRESH` | 12 | Global variance filter threshold |
| `V_LOCAL_THRESH` | 6 | Local variance filter threshold |
| `TILE_SIZE_SW` | 512 | Swin tile size |
| `TILE_SIZE_YOLO` | 640 | YOLO tile size |
| `STRIDE` | 256 | Tile overlap stride |
| `SWIN_FREEZE_EPOCHS` | 10 | Epochs with frozen backbone |
| `SWIN_UNFREEZE_EPOCHS` | 10 | Fine-tuning epochs |
| `FUSION_ARCH` | mlp | Fusion model architecture |
| `FOLD_K` | 5 | Cross-validation folds |

## Improving Performance on Limited Hardware

1. **Tile batching**: Increase `batch_size` in config for GPU; reduce for CPU.
2. **ONNX quantization**: Run `scripts/export_onnx.sh` to quantize FP32 → INT8 (~2-3× speedup on CPU).
3. **Prune model heads**: Remove unused output heads in the fusion model for inference-only deployment.
4. **Pseudo-labeling**: After initial training, run model on unlabeled tiles to generate additional training data with `pseudo_label_threshold: 0.85`.
5. **Strong augmentation + k-fold + ensembling**: Average logits across all k folds at inference time for better accuracy.
6. **Lite inference**: Use `lite_infer.py` to skip GeoSAM and heavy depth on edge devices.
7. **Use the small test config**: `example_configs/small_local_test_config.yaml` for rapid iteration.
